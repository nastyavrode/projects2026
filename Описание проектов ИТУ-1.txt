Влад
Сборщик «мертвых душ» в университетских группах (для старост)
Проблема: Старостам приходится вручную обзванивать/писать 30 человек, чтобы узнать, кто придет на пару или сдаст лабу, а кто нет. Групповые чаты — это хаос.

Решение: Веб-панель, где студенты за 1 клик отмечают свое состояние («буду», «опаздываю», «не приду», «болею»). Староста видит сводку в реальном времени и может одним кликом экспортировать список отсутствующих в Excel.

Технологии: Django (быстрая админка) или Next.js, PostgreSQL, WebSockets, экспорт в XLSX/CSV.

Сережа

 Каталог VPN (сервисов виртуальных частных сетей)
Проблема
Пользователи (особенно новички) тратят часы на выбор VPN, потому что:

Информация разрознена (YouTube-обзоры часто заказные, на форумах — противоречивые отзывы).

Провайдеры скрывают реальные показатели (логируют ли данные, есть ли утечки DNS/WebRTC, настоящая скорость в разных странах).

Сложно сравнить десятки параметров: цена, количество стран, протоколы (WireGuard / OpenVPN), поддержка torrent, Netflix, наличие бесплатной версии и ее ограничения.

Пользователь не может быстро найти VPN для своей конкретной задачи: «играть с низким пингом в EU», «смотреть Hulu», «просто скрыть IP для торрентов».

Решение
Создать агрегирующий каталог-конструктор VPN-сервисов с честным рейтингом на основе открытых данных и краудсорсинга.

Основной функционал:

Фильтр-конструктор (шаг за шагом):

«Какая у вас цель?» → (стриминг / торренты / работа / анонимность)

«Бюджет в месяц?» → (бесплатно / до $3 / $3-6 / $6+)

«Какие страны вам нужны?» → (Россия, США, Нидерланды и т.д.)

«Нужен ли kill switch?», «Важен ли аудит безопасности?»

На выходе — 2-3 подходящих варианта с пояснением почему именно они.

Сравнительная таблица с живыми данными (парсинг + ручные проверки):

Цена за месяц/год, количество одновременных устройств.

Реальная скорость (данные с speedtest-серверов, усредненные за неделю).

Результаты тестов на утечки DNS/WebRTC/IPv6.

Какие регионы Netflix/Disney+/Hulu разблокирует.

Наличие российской регистрации/серверов (красный флаг для многих).

Система «доверия»:

Значок «прошел независимый аудит» (Cure53 и др.).

Значок «no-logs подтверждено судом».

История инцидентов (были ли взломы, передавали ли данные властям).

Пользовательские замеры (краудсорсинг):

Любой пользователь может запустить встроенный speedtest до выбранного сервера и анонимно поделиться результатом (пинг, скорость загрузки/отдачи). Данные агрегируются и показываются рядом с официальными.

Технологии
Компонент	Технологии
Фронтенд	React / Next.js (SSR для SEO, чтобы каталог индексировался поисковиками по запросам «VPN сравнение»)
Таблицы и фильтры	TanStack Table (React Table) + Fuse.js (поиск с опечатками)
Скоростные тесты	Клиентский JS, замеряющий загрузку изображения с сервера (или WebRTC для пинга), либо интеграция с Speedtest Custom
Бэкенд API	Node.js (Express) или Python (FastAPI)
База данных	PostgreSQL (для VPN-провайдеров, параметров, отзывов) + Redis (кэш для страниц каталога)
Сбор данных (парсинг)	Python (BeautifulSoup, Scrapy) — раз в неделю собирать цены и условия с сайтов VPN
Автоматические проверки	Puppeteer (безголовый браузер) для проверки, открывается ли Netflix США с IP данного VPN
Аутентификация	JWT, регистрация через email или OAuth (Google/GitHub)
Фронтент-стейт	Zustand или Redux Toolkit
UI Kit	Tailwind CSS + shadcn/ui (современные компоненты)
Дополнительно:

Демо-режим: можно запустить эмуляцию подключения к VPN через WebSocket-туннель (ограниченный трафик), чтобы пользователь «пощупал» скорость.

Telegram-бот: для быстрого запроса «найди VPN с сервером в ОАЭ и ценой до $4».

Максим

Калькулятор «Реальная стоимость подписки» (с учетом забытых)
Проблема: У среднестатистического человека есть 5-10 подписок (Netflix, Spotify, iCloud, ChatGPT, VPN). Он платит за многие, но не пользуется или забыл отключить пробный период.

Решение: Веб-приложение, куда пользователь добавляет подписки с периодичностью (месяц/год) и указывает, сколько раз за последние 30 дней реально заходил в сервис. Калькулятор показывает: «Вы потратили 3450 руб, из них 2100 — впустую».

Технологии: LocalStorage (чтобы не хранить данные на сервере), React (хуки), визуализация диаграмм (Chart.js), экспорт в PDF-отчет.

Сева+Дима
Карта доступных тренажерных залов, спортплощадок, спортмагазинов, бассейнов
Проблема
Человек, который хочет заняться спортом (или купить инвентарь), сталкивается с информационным хаосом:

Тренажерные залы — скрывают цены, не пишут загруженность по часам, не указывают наличие свободных тренеров, зон кроссфита/воркаута.

Спортплощадки (воркаут, футбол, баскетбол) — на картах Google/2ГИС нет информации, освещены ли они, есть ли сетки, бесплатные ли, не сломаны ли турники.

Бассейны — сложно найти актуальное расписание «свободного плавания» (не дорожки для занятий), цены на абонементы, требования к справкам.

Спортмагазины — нет фильтрации по наличину конкретной модели кроссовок/велосипеда в конкретном магазине рядом с домом.

Итог: человек тратит 2 часа на поиски, едет в зал, а там «сауна не работает, абонемент только на год, сейчас детская секция — взрослых не пускают».

Решение
Создать интерактивную карту с живыми и краудсорсинговыми данными по четырем категориям. Упор на актуальность и детализацию.

Функциональные блоки:

2.1. Карта с фильтрами
Тип объекта: Зал / Площадка / Бассейн / Магазин.

Цена (бесплатно / до 500₽ за разовое / абонемент до 2000₽/мес / премиум).

Режим работы (круглосуточно, утро, вечер).

Доступность для инвалидов (пандус, широкие проходы).

2.2. Детальная карточка каждого места
Для тренажерного зала:

Фото зон (кардио, свободные веса, тренажеры, зона функционала).

График загруженности (зеленый/желтый/красный) на основе истории посещений от пользователей.

Есть ли тренер в зале сейчас (отмечают посетители).

Пробный день — можно ли прийти бесплатно, что нужно (паспорт?).

Для спортплощадки (воркаут/футбол/баскетбол/волейбол):

Состояние покрытия (асфальт, резина, трава — оценка 1-5).

Есть ли освещение (работает ли вечером).

Сломаны ли элементы (пользователи ставят флаг «турник сломан»).

Бесплатно / нужна запись.

Для бассейна:

Расписание свободного плавания (без групп и аквааэробики) — с возможностью экспорта в календарь.

Температура воды (если измеряют пользователи).

Требования к медицинской справке (и где ее делают рядом).

Глубина, есть ли вышка, детская зона.

Для спортмагазина:

Поиск по товару («где есть кроссовки Asics Kayano 31 мой размер 44?»).

Актуальные цены (пользователи фотографируют ценники).

Примерка (есть ли место, чистое ли).

2.3. Краудсорсинг и модерация
Любой пользователь добавляет объект (через клик на карту + форма).

Любой пользователь обновляет статус: «свет не работает», «зал закрыт на ремонт», «цена выросла».

Система доверия: за 5 подтвержденных обновлений — «проверенный участник», его данные отображаются без модерации.

2.4. Умные функции
«Рядом со мной сейчас» — показать тренажерные залы с низкой загруженностью в радиусе 1 км, куда можно прийти через 10 минут.

Маршрут до ближайшей бесплатной воркаут-площадки с учетом времени пешком/на транспорте.

Сравнение бассейнов по стоимости часа и близости к метро.

Технологии
Компонент	Технологии
Картографический движок	Leaflet (слой OpenStreetMap) или Yandex Maps API (лучше для РФ: пробки, детализация). При использовании Leaflet — бесплатно, но нужен тайловый сервер (например, карты от DataHub).
Фронтенд	Vue 3 + Vite (легче, чем React, для картографических приложений) или Svelte (очень быстрый рендеринг маркеров)
Бэкенд	Node.js + Express (просто) или Django + DRF (удобная админка для модерации объектов)
База геоданных	PostgreSQL + PostGIS (оптимально для хранения и запросов «все объекты в радиусе 1 км от точки»)
Кэш тайлов (если Leaflet)	Набор статических тайлов на CDN (или использование готовых — CartoDB, Stamen)
Авторизация	JWT + соцсети (VK ID, Google, Яндекс) — чтобы снизить порог входа для добавления данных
Рейтинг загруженности залов	Агрегация времени прихода пользователей (анонимно). Хранится как гистограмма по дням недели и часам
Уведомления	Email (Nodemailer) + Web Push (через сервис-воркер) для оповещений «в вашем любимом зале сейчас мало людей»
Поиск по магазинам (товары)	Elasticsearch (если много товаров) или тривиальный LIKE-поиск в PostgreSQL на старте
Мобильная адаптация	PWA (можно «установить» на телефон) с офлайн-кэшем просмотренных карточек
UI	Tailwind CSS + DaisyUI (для быстрых карточек и форм)
Продвинутый технический трюк для загруженности залов:

Не просить пользователей ставить оценку загруженности (они забывают). Вместо этого — трекер на основе времени открытия карточки зала: если много пользователей смотрят карточку зала в 19:00 и нажимают кнопку «построить маршрут» — система предполагает высокую загруженность.

Этапы реализации MVP (минимально жизнеспособный продукт):

Карта с ручным добавлением 50 объектов (для стартового наполнения).

Форма добавления и базовая модерация (один админ).

Фильтры по типу и цене.

Краудсорсинговые флаги («закрыто», «цена изменилась»).

(Дополнительно) Экспорт в Telegram-бота для быстрых запросов.
