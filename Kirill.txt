
## Общая концепция

**Название (условное):** PyStart: Python для начинающих  
**Цель:** Дать базу по синтаксису и логике Python через мини-уроки и интерактивные задания.  
**Ключевая механика:** Каждая тема разбита на два обязательных шага:  
1. **Короткая теория** (текст + примеры кода).  
2. **Практика** (задания с проверкой).  
После успешного выполнения практики тема считается пройденной и открывается следующая.  

---

## Структура экранов (основные)

### 1. Экран приветствия / Онбординг
Появляется при первом запуске.  
**Что содержит:**
- Краткое описание приложения (3–4 слайда с кнопкой «Далее»).
- Информация о том, что обучение идёт по цепочке «теория → практика».
- Запрос на разрешение отправлять уведомления (напоминания о занятиях).
- Кнопка «Начать обучение» → переход к экрану выбора темы / первой теме.

### 2. Главный экран (Дэшборд)
Показывает прогресс пользователя.  
**Элементы:**
- Приветствие: «Привет, [Имя]» (имя запрашивается при первом входе).
- Круговая диаграмма / прогресс-бар: сколько тем пройдено из общего числа.
- Список разделов (тем) в виде карточек:
  - Переменные и типы данных
  - Условные операторы (if/elif/else)
  - Циклы (for, while)
  - Списки
  - Словари
  - Функции
  - Обработка ошибок (try/except)
  - (Опционально) Работа с файлами
- У каждой карточки статус:
  - 🔒 Заблокировано (пока не пройдена предыдущая)
  - 📘 Доступна (теория не пройдена)
  - ✏️ Теория пройдена, практика ждёт
  - ✅ Тема завершена
- Кнопка «Продолжить обучение» (ведёт к следующей незавершённой теме).

### 3. Экран теории по теме
Открывается при нажатии на карточку темы.  
**Содержимое:**
- Заголовок темы (например: «Переменные и типы данных»).
- Текст теории (разбит на короткие абзацы, до 5–7 предложений).
- Вставки с кодом (форматированы как `code_block`).
- Примеры использования (можно с возможностью копирования).
- Иллюстрации (схемы, иконки) для наглядности.
- Кнопка внизу: «Понятно, перейти к практике» (активируется после скролла до конца или по нажатию).
- Кнопка назад (сохраняет прогресс прочтения теории).

### 4. Экран практики по теме
Проверяет усвоение материала.  
**Варианты форматов заданий (в зависимости от темы):**
- **Выбор правильного ответа** (тест с 1 или несколькими вариантами).
- **Восстановить порядок** (перетащить строки кода в правильном порядке).
- **Найти ошибку** (показан код с багом, нужно выбрать исправление).
- **Что выведет этот код?** (вывод в консоль).
- **Ввод ответа** (число, строка, True/False).

**Общие элементы:**
- Условие задачи (текст + пример кода, если нужно).
- Поле для ответа (или варианты выбора).
- Кнопка «Проверить».
- Обратная связь:
  - ✅ Правильно → переход к следующему заданию (или завершение темы).
  - ❌ Неправильно → показывается пояснение и кнопка «Попробовать снова».
- Индикатор прогресса по заданиям (задание 1 из 3).

После выполнения всех заданий темы — экран поздравления и кнопка «Следующая тема».

### 5. Экран с редактором кода (для продвинутых тем)
Используется для практики по темам «Функции», «Списки», «Циклы» (чтобы пользователь писал небольшой код сам).  
**Элементы:**
- Условие задачи сверху.
- Редактор с подсветкой синтаксиса (можно упрощённый).
- Кнопка «Запустить».
- Область вывода (консоль) под редактором.
- Кнопка «Проверить решение» (сравнение ожидаемого и фактического вывода).

### 6. Экран профиля
- Статистика: всего тем, пройдено тем, процентов успешных решений.
- Настройки: 
  - Включить/выключить уведомления.
  - Сменить тему (тёмная/светлая).
  - Сбросить прогресс.
- Кнопка «Поделиться приложением».

### 7. Экран достижений (геймификация)
Появляется по мере прохождения тем.  
**Что содержит:**
- Значки (ачивки) за:
  - Первую пройденную тему.
  - 5 тем подряд без ошибок.
  - Написание первого работающего кода в редакторе.
- Показывается, какие значки уже получены.

## Пример прохождения одной темы (на примере «Переменные»)

1. Пользователь на главном экране нажимает карточку «Переменные и типы данных».
2. Открывается экран теории:
   - Текст: «Переменная — это имя, которое ссылается на значение. В Python не нужно указывать тип...»
   - Примеры: `x = 5`, `name = "Анна"`, `is_student = True`.
   - Типы данных: int, float, str, bool.
   - Кнопка «К практике».
3. Экран практики:
   - **Задание 1:** Какая переменная хранит строку? (варианты: `age = 25`, `name = "John"`, `is_valid = False`).
   - **Задание 2:** Что будет выведено? `a = 10; b = 3; print(a + b)` → варианты: 13, 103, ошибка.
   - **Задание 3 (если есть редактор):** Создайте переменную `city` со значением "Москва" и выведите её.
4. После всех правильных ответов — поздравление, тема помечается ✅, открывается следующая тема («Условные операторы»).

## Дополнительные экраны (опционально)

### 8. Экран подсказок
- Доступен на экране практики (кнопка «Подсказка»).
- Показывает не прямой ответ, а направление (например: «Вспомни, какой оператор проверяет равенство»).

### 9. Экран «Мои ошибки»
- Собирает задания, в которых пользователь ошибся.
- Можно перепройти их отдельно.

### 10. Экран быстрого повторения (карточки)
- Показывает случайные вопросы из пройденных тем в стиле флеш-карт.

## Технические рекомендации для разработки

- **Платформа:** React Native / Flutter (для кроссплатформенности).
- **Хранение данных:** SQLite (прогресс, ответы пользователя) + локальное хранилище.
- **Редактор кода:** Можно использовать компонент `Monaco Editor` (для WebView) или упрощённый `react-native-code-editor`.
- **Проверка кода:** Выполнять код безопасно (песочница) через встроенный интерпретатор Python (например, `Skulpt` для клиентской проверки или отправка на бэкенд).
- **Офлайн-режим:** Вся теория и задания должны работать без интернета (кроме возможного расширенного редактора).

## Путь пользователя (User Journey)

1. Установил → прошёл онбординг.
2. Увидел первую тему «Переменные» → прочитал теорию.
3. Выполнил 3 задания → получил ✅.
4. Перешёл к теме «Условные операторы» → повторил цикл.
5. Зашёл в профиль → увидел прогресс 25%.
6. Получил уведомление: «Давно не занимались? Продолжите с темы „Циклы“».
7. Через неделю завершил все темы → получил финальный сертификат (можно встроенный, сгенерированный в приложении).
